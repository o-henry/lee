# Let's get started

* UI/UX 디자인을  식별 및 이해합니다.
* 하드코딩을 통해 스켈레톤 코드를 우선 구현합니다.
* 스크린 과 컴포넌트를 구분합니다. 

> 앞으로의 포스팅에서는 figma를 통해 작성한 UI를 토대로 하여 개발합니다.
  또한 중복된 요소의 경우 분리합니다. 이는 디자인 시스템의 도입을 생각할 수 있게합니다.

# Feature
1. 기존 SNS의 기능에 한가지 규칙을 추가합니다.
2. 업로드 가능한 케이스
    * 앱을 통해 촬영한 사진
    * 사진 metadata가 현재 시간 기준 하루이내인 경우 업로드 가능
    * 사용자 위치가 사진 metadata의 위치정보와 일치하는 경우
    * 업로드 이후 하루가 지나면 개인 피드에만 저장되고 따로 메인에 노출시키지 않습니다.
3. 같은 위치에 다른 사진이 업로드 된 경우 해당 사진을 피드의 가장 맨 위에 노출시킵니다.
4. 필터링을 추가합니다.
5. 같은 사용자가 여러번 업로드 하는 경우 가장 최신 피드만을 메인에 로드합니다.
